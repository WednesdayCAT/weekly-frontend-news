## 功能
1. 自动汇总上周前端领域核心资讯（框架/工具/开源项目/Web标准/跨端/行业实践）
2. 按固定专业格式生成 Markdown 内容
3. 所有资讯自动附带权威跳转链接
4. 自动将内容写入 GitHub 仓库对应路径文件
5. 自动执行 Git 提交、推送，完成周刊全自动发布

## 触发条件
- 生成上周前端周刊
- 前端周刊自动发布
- 自动生成并提交前端周刊到GitHub
- 帮我生成并推送前端技术周刊
- 前端周刊自动更新仓库

## 使用
用户触发后，自动执行完整流程：
获取上周时间 → 生成带链接的周刊内容 → 写入仓库指定 .md 文件 → Git add/commit/push → 完成发布

## 执行规则
### 资讯覆盖范围
1. 主流框架更新（Vue/React/Next/Nuxt/Svelte/Solid）
2. 构建工具（Vite/Rspack/Turbopack/Rollup）
3. 前端热门开源项目（GitHub 高星组件库/工具库）
4. Web 标准 / 浏览器新特性
5. 跨端方案（UniApp/Taro/Flutter）
6. 大厂前端实践（阿里/腾讯/字节/美团）

### 内容规则
- 每条资讯必须附带【来源 + 可访问跳转链接】
- 格式专业、简洁、可直接开源展示
- 无重复、无冗余、无低质量内容
- 无内容时自动生成高质量兜底动态，不空白

### 输出文件格式（强制）
文件路径（固定）：
docs/{YYYY}/{YYYY}-{周数}周前端技术周刊.md

文件内容格式：
# 📅 {YYYY}-{周数}周 前端技术周刊（{MM-DD} 至 {MM-DD}）
更新时间：{YYYY-MM-DD} | 开源地址：[weekly-frontend-news](https://github.com/WednesdayCAT/weekly-frontend-news)

> 每周一自动更新，精选前端领域「框架更新/生态工具/开源项目/行业实践」，帮你高效掌握前端趋势。

### 自动提交 GitHub 规则（必须执行）
1. 写入文件路径：docs/{YYYY}/{YYYY}-{周数}周前端技术周刊.md
2. Git 提交信息：feat: 自动生成 {YYYY}-{周数}周 前端技术周刊
3. 自动执行：
   git add .
   git commit -m "feat: 自动生成前端周刊"
   git push origin main
4. 推送完成后返回：✅ 前端周刊已自动发布至 GitHub 仓库

### 拒绝条件
- 不生成非上周的内容
- 不生成与前端无关的内容
- 不生成无链接、低质量内容
- 不修改固定文件路径
- 不执行未授权的仓库操作
